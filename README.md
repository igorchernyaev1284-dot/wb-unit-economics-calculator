# AI-Аналитик Юнит-Экономики Wildberries (n8n + Python + LLM)

Автоматизированное production-ready решение для еженедельного финансового аудита и расчета юнит-экономики селлеров Wildberries. Сценарий обрабатывает сырые детализированные отчеты маркетплейса, агрегирует данные с помощью Python и передает их ИИ-агенту для формирования бизнес-аналитики.

## 💼 Бизнес-ценность решения

* **Исключение ручного труда:** Полностью заменяет ручной разбор многотысячных Excel-строк финансовым аналитиком или менеджером.
* **Предотвращение кассовых разрывов:** Автоматически подсвечивает нерентабельные SKU (товары с отрицательной маржой из-за логистики и возвратов).
* **Скорость и фокус:** Руководитель получает сухой финансовый аудит и готовый план действий в Telegram через секунды после загрузки отчета.

## ⚙️ Архитектура и логика работы

Сценарий реализован в платформе автоматизации **n8n** и состоит из следующих этапов:

1. **TG: Входящий файл** — Telegram-бот принимает еженедельный детализированный отчет Wildberries в формате `.xlsx`.
2. **XLSX: Парсинг отчета** — нода извлекает бинарные данные и преобразует их в JSON-массив.
3. **PY: Агрегация данных** (код в `calculator.py`) — встроенный Python-скрипт группирует транзакции по SKU, сопоставляет их с базой себестоимости (COGS), рассчитывает общую логистику, возвраты, выплаты, чистую прибыль и маржинальность.
4. **AI: Анализ экономики** — LLM-агент (Claude / GPT) проводит жесткий финансовый аудит агрегированного отчета, выявляет "дойных коров" и убыточные позиции.
5. **TG: Отправить отчет** — отправка структурированного HTML-отчета с планом действий в рабочий Telegram-чат.

## 🛠 Технический стек

* **Платформа:** n8n (v2 Code Node)
* **Язык обработки данных:** Python 3 (Native)
* **Интеграции:** Telegram Bot API, OpenRouter API (LLM)
* **Формат данных:** JSON, XLSX

## 📂 Содержимое репозитория

* `calculator.py` — чистый Python-код для агрегации финансовых показателей Wildberries внутри n8n.
* `wb_workflow.json` — готовый шаблон сценария для импорта в n8n.

## 🚀 Инструкция по запуску

1. **Импорт сценария:** Создайте новый workflow в n8n, нажмите `Ctrl + F` (или меню в правом верхнем углу) и выберите **Import from file**, указав файл `wb_workflow.json`.
2. **Настройка Python-ноды:** Откройте ноду `PY: Агрегация данных` и пропишите себестоимость ваших товаров в словаре `cogs_dict` (вместо тестовых значений).
3. **Подключение API:** Настройте учетные данные для Telegram-триггера и ноды отправки сообщений, а также API-ключ для ИИ-модели в ноде `Модель Агента`.
